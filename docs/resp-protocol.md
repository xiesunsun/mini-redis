# RESP 协议

## 概述
RESP 全称是 REdis Serialization Protocol,它主要是用于Redis客户端与Redis服务端之间的通讯

## 数据类型
支持五种数据类型：
1. 简单字符串  
符号：+
用途：成功响应，比如+OK
2. 错误 
符号：-
用途：错误响应，比如-ERR wrong type
3.整数 
符号：：
用途：数字响应，比如：1
4. 批量字符串
符号：$
用途：普通字符串
5. 数组
符号：*
用途：命令或多值
注意每个部分都以 \r\n 结尾

## 请求格式
Redis 客户端发送任何命令，都是用统一使用数组的格式进行封装，数组里的每个元素都是批量字符串
具体的格式如下所示
```
*元素个数\r\n
<元素1>
<元素2>
……
```
具体的格式序列如下所示
(*+数字+\r\n,后面紧跟对应数量的元素)
其中批量字符串的格式如下：
```
$长度\r\n
实际内容
\r\n
```
具体的序列格式如下所示
（$+字节长度+\r\n+内容+\r\n）
以SET name Alice为例进行解释
其中客户端发送的序列如下所示
```
*3\r\n$3\r\nSET\r\n$4\r\nname\r\n$5\r\nAlice\r\n
```
*3\r\n -> 数组开头:这是一个包含3个元素的数组
$3\r\nSET\r\n -> 批量字符串：第一个元素，长度为3个字节，内容是命令名称SET
$4\r\nname\r\n -> 批量字符串：第2个元素，长度是4个字节，内容是键名name
$5\r\nAlice\r\n -> 批量字符串：第三个元素，长度5个字节，内容值是Alice

## 响应格式
所有响应都必须以\r\n结尾
服务器返回的类型与上述RESP所具有的五大类型一致
1. 简单字符串
开头字符：+
完整格式示例：
+OK\r\n
+PONG\r\n
2. 错误
开头字符：-
完整格式示例：
-ERR unknow command 'asdf'\r\n
-WRONGTYPE Operation against a key holding the wrong kind of value \r\n

3. 整数
开头符号：:
完整格式示例：
:1\r\n
:0\r\n
:1000\r\n
4. 批量字符串
开头符号：$
完整格式示例：
$5\r\nAlice\r\n
$0\r\n\r\n
$-1\r\n
5. 数组
开头符号：*
完整格式示例：
*2\r\n$5\r\nhello\r\n$5\r\nworld\r\n
*0\r\n
*-1\r\n

## 特殊值

**nil（空值）**
用批量字符串表示：
$-1\r\n

**空数组**
*0\r\n

**空字符串**
$0\r\n\r\n
